# Veggie Classification

On-device vegetable classifier for the SnapChef fridge assistant. A MobileNetV2 (α=0.35, 96×96, INT8) recognises **9 vegetables + Unknown** from an OV3660 camera attached to a XIAO ESP32S3 Sense. Inference runs at ~270 ms per frame, fully offline, no cloud.

Classes: `Bellpepper, Broccoli, Cabbage, Carrot, Eggplant, Garlic, Onion, Potato, Tomato, Unknown`

---

## Directory layout

```
veggie_classification/
├── data/               # Training data (input)
├── train/              # Notebooks that train and export models
├── model/              # Frozen model snapshots ready to drop into firmware
└── deploy/             # ESP32 / Grove Vision AI sketches
```

### `data/`
Source images and videos used by the training notebooks.

| Path | What |
|---|---|
| `data/huggingface/Train/<Class>/` | The HuggingFace fruit & vegetable dataset, ~80–150 images per class. 52 classes total — only 9 are used as positives, the rest are sampled into the Unknown class. |
| `data/huggingface/val/<Class>/` | Held-out validation split, 10–17 images per class. |
| `data/videos/` | User-recorded clips (`tomato_1.mp4`, `bellpeper_1.mp4`, …). Filename prefix encodes the class. Frames are extracted into the train split by Cell 2b of the V3 notebook. Add `unknown_*.mp4` clips to extend the Unknown class with kitchen-specific OOD (cups, hands, packaging). |
| `data/unknown/` *(optional)* | Drop loose JPEG/PNG stills here for the Unknown class. Cell 2c picks them up. |

### `train/`
Each `classification_V<n>/` is a self-contained training run. Open the `.ipynb` in Jupyter / VS Code and run top-to-bottom.

| Folder | Architecture | Status |
|---|---|---|
| `classification_V0/` | Tiny food classifier (Keras, baseline) | superseded |
| `classification_V1/` | YOLOv8n-cls (Ultralytics) | superseded |
| `classification_V2/` | MobileNetV2-0.35 @ 96×96, 10 classes incl. Cucumber | superseded |
| **`classification_V3/`** | **MobileNetV2-0.35 @ 96×96, 9 veggies + Unknown, video-frame aug, mild colour jitter** | **current** — 86% val acc, 91% confident-only acc |
| `sensecraft/` | `compile_vela.py` — wraps `ethos-u-vela` to compile the Vela-ready TFLite for Grove Vision AI V2 (Himax WiseEye2 + Ethos-U55 NPU). |

Inside `classification_V3/`:
- `mnv2_classification.ipynb` — the training notebook
- `mnv2_dataset/` — auto-built train/val tree (output of Cells 2 / 2b / 2c / 2d)
- `runs/veggie_mnv2/` — checkpoints (`best_head.keras`, `best.keras`), confusion matrix, training curves
- `veggie_mnv2.tflite` — INT8, baked `preprocess_input`, for ESP32 TFLite Micro
- `veggie_mnv2_sensecraft.tflite` — uint8 I/O, for SenseCraft web upload
- `veggie_mnv2_sensecraft_int8.tflite` — int8 I/O, for `vela` compilation
- `model_data.h`, `labels.h` — C headers generated by Cell 13, copy these into a deploy folder

### `model/`
Frozen snapshots — copies of `model_data.h` + `labels.h` corresponding to a training generation. Use these as the canonical source when wiring up a new firmware variant.

| Folder | Source |
|---|---|
| `model/V0/` | classification_V2 (10 classes incl. Cucumber) |
| `model/V1/` | classification_V3 (9 veggies + Unknown) — current |

### `deploy/`
ESP32 / Grove sketches that consume `model_data.h` + `labels.h` and run inference on the device.

| Folder | Target / purpose |
|---|---|
| `esp32_deploy_V0/`, `V1/` | Early YOLO-cls firmware — superseded |
| `esp32_deploy_V2/`, `V3/` | Grove Vision AI V2 (SSCMA) — model lives on the Vision AI module flash, sketch only forwards results |
| `esp32_deploy_V4/` | TFLite Micro on XIAO ESP32S3 with WiFi web preview (commented out) |
| `esp32_deploy_V5/` | TFLite Micro + SSD1306 OLED, three-tier smoothing, no WiFi |
| **`esp32_deploy_V6/`** | **Display-agnostic refactor of V5** — single hook function (`displayResult()`) for any display. **Default sketch — copy + edit this for new projects.** |
| `esp32_grove_serial/` | Grove Vision AI V2 + companion ESP32 sketch that reads SSCMA results over UART |

## Quick start — deploy V6 on a XIAO ESP32S3 Sense

1. **Hardware**: XIAO ESP32S3 Sense (with the camera FFC expansion) + OV3660 module plugged into the FFC connector.
2. **Arduino IDE board manager**: `esp32` by Espressif, **v3.x**.
3. **Library Manager**: install `TensorFlowLite_ESP32`.
4. **Board settings**:
   - Board: `XIAO_ESP32S3`
   - PSRAM: `OPI PSRAM` *(required — the tensor arena lives in PSRAM)*
   - Partition: `Huge APP (3MB No OTA / 1MB SPIFFS)` *(model is ~625 KB)*
5. Open [deploy/esp32_deploy_V6/esp32_deploy_V6.ino](deploy/esp32_deploy_V6/esp32_deploy_V6.ino) and flash.
6. Open Serial Monitor at 115200 baud — you should see lines like:

   ```
   [273ms] raw:Tomato 84.2%  [snap]  -> Tomato (84.2%)
   [275ms] raw:Tomato 81.0%  [snap]  -> Tomato (81.0%)
   ```

To use a different display (OLED / LCD / NeoPixel / MQTT), edit only the `displayResult()` function near the bottom of the sketch — `setup()`, `loop()`, the inference pipeline, and the smoothing logic stay the same. The `ClassificationResult` struct exposes everything you need: `label`, `confidence`, `inferenceMs`, `tierName`, `rawConfidence`, etc.

## Quick start — train your own model

1. Add training data:
   - Drop class folders into `data/huggingface/Train/<Class>/` and `data/huggingface/val/<Class>/`, **or** use the bundled HuggingFace dataset if you already have it.
   - *(Optional)* Record `<class>_<n>.mp4` clips (~30 s each) and drop them into `data/videos/`. Cell 2b extracts ~30 frames from each into the train split.
   - *(Optional)* Drop kitchen-specific OOD images into `data/unknown/` (cups, hands, packaging). Cell 2c bundles them into the Unknown class along with a sample of unused HuggingFace classes.
2. Open [train/classification_V3/mnv2_classification.ipynb](train/classification_V3/mnv2_classification.ipynb) and run cells in order. Notable cells:
   - **Cell 2 / 2b / 2c**: build `mnv2_dataset/` from HuggingFace + videos + Unknown sources.
   - **Cell 2d**: TF-decode sweep — auto-removes any image Keras can't decode (catches WebP-disguised-as-JPEG and other corrupt files). Re-run any time training crashes with "Unknown image file format".
   - **Cell 4**: dataset pipeline + augmentation. Tunable: `random_hue_sat()` strength.
   - **Cell 5**: model build (MobileNetV2 + GAP + Dropout + Dense).
   - **Cell 6**: Phase 1 training (head only, frozen backbone). The `EarlyStopping(restore_best_weights=True)` keeps the best-val-acc checkpoint.
   - **Cell 7**: Phase 2 fine-tuning. **Disabled by default** (`EPOCHS_FINETUNE = 0`) — on this dataset Phase 2 overfits and hurts. Re-enable if you have substantially more data.
   - **Cell 9 / 10**: evaluate, confusion matrix, per-class precision/recall.
   - **Cell 11 / 11b / 11c**: export 3 TFLite variants (ESP32 INT8, SenseCraft uint8, Vela int8).
   - **Cell 13**: write `model_data.h` + `labels.h`.
3. Copy the generated `model_data.h` and `labels.h` into your deploy folder of choice (e.g. `deploy/esp32_deploy_V6/`) and re-flash.

For the Grove Vision AI V2 path, additionally run `python train/sensecraft/compile_vela.py train/classification_V3/veggie_mnv2_sensecraft_int8.tflite` and upload the resulting `*_vela.tflite` via the SenseCraft web UI.

## Model details (V3 / current)

- Architecture: MobileNetV2, width multiplier α = 0.35
- Input: 96×96 RGB, [0, 255]; `preprocess_input` ((x/127.5)−1) baked into the graph
- Quantisation: INT8 weights + activations, calibrated on 200 train samples
- TFLite size: ~625 KB; tensor arena: 512 KB
- ESP32S3 inference: ~270 ms / frame
- Confidence threshold: 0.6 (sub-threshold predictions render as "Object not known")
- Validation accuracy: 0.86 overall, 0.91 on confident-only predictions
